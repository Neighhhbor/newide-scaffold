# F 方向埋点清单（RFC §1 记忆纵向 + §2 长程协调 + §3 Council 互决策消融 + §4 端到端打榜）

## 共用定义

**埋点**：凡是为 F 方向指标提供**原始信号**的记录点（Harness 输出、`EventStore` 事件、嵌套字段、持久化实体等），**不限于** `event_type`。

| 层级           | 含义                                                              |
| -------------- | ----------------------------------------------------------------- |
| **L1 Harness** | F 组评测外层输出（不要求 scaffold 落盘）                          |
| **L2 Event**   | `EventStore.appendEvent()` 独立事件                               |
| **L3 Field**   | 嵌在 L2 payload、对象实体或 Spec 存储里的字段（不必单独成 event） |

**L2 信封**（实现以 scaffold 为准；C Spec 另增可选 `subject_type`）：

`event_id`, `event_type`, `subject_id`, `run_id`, `task_id`, `payload`, `created_at`, `schema_version`

**依据**：

- F 方向 RFC §1–§2（评测指标）
- §1 字段：`方向B：Agent角色与记忆系统——Spec`
- §2 事件与对象：`方向C：协调层 状态管理 与 council Spec`
- 实现锚点：`newide-scaffold`（F 观测层：`src/telemetry/`；用法见同目录 `README.md`）

**观测出口**：F 方向信号写入 `TelemetrySink`（默认 `NoopTelemetrySink`）。C 侧 EventStore 事件经 `RuntimeOrchestrator.appendEvent` 自动 mirror；B 侧 memory-cycle 经 `recordMemoryCycleTelemetry` 观测；L1 由外部 Harness 经 `FHarnessTelemetryPort` 写入。**不替代** B/C 的 EventStore emit 与实体持久化。

**命名边界**：

- §1 的 B Spec 是**实体字段 / 数据流契约**，不是 EventStore 事件名表；`memory.*`、`metrics.updated` 属于 F 评测 / scaffold 的**拟议观测事件名**。
- `memory.ablation` 属于 F Harness 实验配置键，用于承载 RFC §1.2 的 B0 / B1 / B2 / B3 消融条件；不是 B Spec 字段。
- 若 B 方向正式补充事件契约，应优先以 B 事件契约为准；当前清单只约束 F 方向需要观察哪些原始信号。
