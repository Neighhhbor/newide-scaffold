## Demo 修改计划

### 执行流程

```
创建 3 个 Agent（前端/后端/运维）
   ↓
collectCompetitionClaims(CSS 任务)        ← Agent 自评，只打印参选者
   ↓
选中第一个参选者 → dispatchTask(role_id)  ← 使用自定义 mock deps
   │                                        （提取/晋升用空实现，不调 LLM）
   ↓
主动 processAgentBuffer(memory, seq, llm) ← 真实 LLM 提取 + 晋升
   ↓
打印提取/晋升结果
```

### 需要改什么

**1. 自定义 mock DriverReturn** — 用真实感的中文 CSS 修复报告：

```typescript
const mockDriverReturn: DriverReturn = {
  summary: '成功修复登录页面 CSS 布局问题',
  artifacts: [
    { type: 'file', path: 'src/pages/Login.css', summary: '新增移动端响应式样式' },
    { type: 'file', path: 'src/pages/Login.tsx', summary: '调整表单容器结构' },
  ],
  decisions: [
    {
      point: '布局方案选择',
      options: ['Flexbox', 'CSS Grid', 'float'],
      chosen: 'Flexbox',
      reason: '兼容性好，一维布局更适合表单场景',
    },
  ],
  blockers: [],
  referenced_experiences: [],
  assumptions: [
    {
      assumption: '目标浏览器支持 Flexbox',
      risk_if_wrong: '需添加 autoprefixer 或回退方案',
    },
  ],
};
```

**2. 自定义 deps** — pipeline 中用这个 mock driver，提取器/晋升器用空实现（不消耗 LLM）：

```typescript
const pipelineDeps: AgentRunDeps = {
  queryMemory: async () => ({ experiences: [], skills: [] }),
  planTaskInstruction: async (task) => task.spec,
  invokeDriver: async () => mockDriverReturn,
  extractor: { extract: async () => ({ experiences: [], result: { ... } }) },
  promote: async () => ({ check: { eligible: false, ... } }),
  contextCleaner: { clean: async () => null },
};
```

**3. dispatchTask 后用 `processAgentBuffer`** 手动触发 LLM 提取+晋升：

```typescript
const memory = createAgentMemoryScope(repo, buf, role_id);
const seqs = await memory.listPendingBufferSeqs();
for (const seq of seqs) {
  const { extraction, promotion } = await processAgentBuffer(memory, seq, task, llm);
  // 打印结果...
}
```

**4. 5 个步骤分别打印：**

- Step 1: 创建 Agent
- Step 2: 竞标收集 → 展示参选者
- Step 3: 派发 → 展示 mock driver 返回
- Step 4: 主动提取+晋升 → 展示 LLM 中文结果
- Step 5: 存储状态
