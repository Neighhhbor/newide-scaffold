## Prompt 集中管理方案

### 问题

当前 5 个 prompt 散落在 5 个文件中，管理方式不一致（有的抽了 const，有的 inline），**最关键的两个 prompt**（顶层 agent 系统提示 / Driver 指令规划）还是静态文本，缺少从 Repository 读取画像动态组装的能力。

### 方案

新建 `src/memory/prompts/` 目录，用 `.ts` 文件（非 `.txt`，避免构建配置）集中存放 prompt 模板，适配器从 prompts/ 导入。

```
src/memory/prompts/
├── index.ts                        ← 统一导出
├── agent-system-prompt.ts          ← 🆕 顶层 agent 系统提示构建器（含 persona 注入）
├── planner.ts                      ← 📦 任务指令规划器 system prompt (从 llm-task-instruction-planner 移出)
├── context-cleaner.ts              ← 📦 上下文清理 system prompt (从 context-cleaner 移出)
├── experience-extractor.ts         ← 📦 经验提取 system prompt (从 llm-experience-extractor 移出)
└── skill-promotion.ts              ← 📦 技能晋升 system prompt (从 llm-skill-promotion 移出)
```

### 核心改动：buildAgentSystemPrompt

```typescript
// prompts/agent-system-prompt.ts
export async function buildAgentSystemPrompt(
  memory: AgentMemoryScope,
  toolDefinitions: ToolDefinition[],
): Promise<string> {
  const persona = await memory.getPersona();
  const skills = await memory.listSkills();
  const metrics = await memory.getMetrics();

  return [
    `You are Agent "${persona.role_id}".`,
    '',
    `## Your Identity`,
    `Summary: ${persona.summary}`,
    `Skills: ${persona.skills_overview}`,
    `Experience: ${persona.experience_coverage}`,
    `Performance: ${persona.recent_performance}`,
    '',
    `## Your Skills (${skills.length})`,
    ...skills.map((s) => `[${s.id.slice(0, 8)}] ${s.description}`),
    '',
    `## Available Tools`,
    ...toolDefinitions.map((def) => `- ${def.function.name}: ${def.function.description}`),
    '',
    `## Behavior Rules`,
    `- Use query_memory to check relevant past experiences and skills before acting.`,
    `- Use invoke_driver to dispatch concrete sub-tasks to the Driver Agent.`,
    `- Keep track of what the driver returns and use it to inform next steps.`,
    `- When the task is complete, summarize the result clearly with "[done]".`,
  ].join('\n');
}
```

| `prompts/context-cleaner.ts`               | 🆕   | 导出 `CONTEXT_CLEANER_SYSTEM_PROMPT` 常量                           |

### 不改动的内容

- **Builder 函数**（`buildCleanPrompt`, `buildExtractionPrompt`, `buildPromotionPrompt`）— 含有数据格式化逻辑，留在原 adapter
- **工具描述**（`query-memory-tool.ts`, `invoke-driver-tool.ts` 的 `description` 属性）— 类属性，不属于「prompt 模板」
- **任务完成启发式检查**（`agent.ts` 的 `isTaskComplete` 数组）— 验证逻辑，不属于 prompt
- **测试 prompt**（api-smoke.ts）— 测试数据
- **`AgentToolConfig.systemPrompt`** — 仍可传入自定义 prompt 覆盖 builder
