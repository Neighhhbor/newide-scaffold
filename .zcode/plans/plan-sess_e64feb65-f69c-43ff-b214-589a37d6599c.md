## 计划：集成测试 + DeepSeekToolCallingClient 适配器

### 目标

编写一个端到端集成测试，使用**真实 DeepSeek API** 驱动 agent 的 tool-calling 循环，验证完整的：
`submitTask → executeTask → LLM 决策 → invoke_driver → 结果分析 → writeToBuffer` 链路。

### 前置：补齐适配层（关键）

当前 `DeepSeekLlmClient` 只实现了 `LlmClient`（`.complete()`），而 Agent 的 tool-calling 模式需要 `ToolCallingClient`（`.completeWithTools()`）。需要新建一个适配器。

**新文件：** `src/memory/adapters/deepseek-tool-calling-client.ts`

实现逻辑：

- 复用现有 `DeepSeekLlmClient` 的 API key / model / baseUrl 配置模式
- 将 `ToolCallMessage[]` 转换为 OpenAI 兼容的消息格式（system/user/assistant/tool 角色）
- 将 `ToolDefinition[]` 作为 `tools` 参数发送
- 解析响应：若含 `tool_calls` 则返回 `ToolCallResult`，若只有文本则返回 `content`
- DeepSeek API 是 **OpenAI 兼容** 的，原生支持 function calling

### 集成测试

**新文件：** `src/memory/test/integration/agent-loop-integration.test.ts`

#### 测试剧情（单轮、简单）

```
任务：“请执行一个简单的 shell 命令：输出 'Hello Agent Loop' 到 greeting.txt 文件”
                                         │
                    ┌─────────────────────┴────────────────────┐
                    │  Agent (真实 DeepSeek LLM)                │
                    │  第1轮：LLM 分析任务 → 决定调用           │
                    │    invoke_driver({ instruction: "..." })  │
                    │  第2轮：看到 driver 返回成功 →            │
                    │    总结结果 → "Task completed. [done]"    │
                    └─────────────────────┬────────────────────┘
                                         │
                    ┌─────────────────────┴────────────────────┐
                    │  Mock DriverHandler                      │
                    │  接收到任何 instruction → 返回固定的      │
                    │  DriverReturn (满足 Zod 契约)             │
                    └─────────────────────┬────────────────────┘
                                         │
                    ┌─────────────────────┴────────────────────┐
                    │  finalizeLoop → writeToBuffer            │
                    │  buffer 中有一条 pending 记录             │
                    └──────────────────────────────────────────┘
```

#### Mock DriverReturn（完全满足 DriverReturnSchema 契约）

```typescript
{
  artifacts: [{
    type: 'file',
    path: 'greeting.txt',
    summary: 'Created greeting.txt with "Hello Agent Loop"',
  }],
  summary: 'Successfully executed shell command: greeting.txt has been created.',
  decisions: [{
    point: 'Output file creation',
    options: ['Use echo command', 'Use printf'],
    chosen: 'Use echo command',
    reason: 'Simplest approach for a single-line output',
  }],
  blockers: [],
  referenced_experiences: [],
  assumptions: [{
    assumption: 'Target directory is writable',
    risk_if_wrong: 'File creation would fail with permission error',
  }],
}
```

#### 验证点

| #   | 验证                                  | 断言                                                       |
| --- | ------------------------------------- | ---------------------------------------------------------- |
| 1   | 任务完成                              | `result.status === 'completed'`                            |
| 2   | LLM 实际调用了 invoke_driver          | mock handler 被调用次数 >= 1                               |
| 3   | buffer 有 pending 条目                | `meta.pending_count === 1`                                 |
| 4   | buffer snapshot 的 driver_return 正确 | `snapshot.driver_return.summary` 包含预期文本              |
| 5   | 提取/晋升未同步执行                   | `result.cycle.extraction.result.experiences_created === 0` |
| 6   | Agent 已休息                          | `agent.getState() === 'sleeping'`                          |

#### 测试保护

- 使用 `it.skipIf(!process.env.DEEPSEEK_API_KEY)` — 无 API key 时自动跳过
- 复用 `.env` 文件中已有的 API key
- 指定 model: `deepseek-chat`（与现有 deepseek-llm-client 一致）
